1. dependencies vs blockedBy Fields

  Current state: Both fields exist but purpose is unclear.

  Recommendation: Remove dependencies field and use only blockedBy.

  Reasoning:
  - blockedBy is the source of truth (explicitly set when creating tasks)
  - dependencies would be the reverse (computed from blockedBy)
  - Keeping both creates duplication and sync issues
  - The blocks field already tracks reverse relationships
  - Current code already computes blocks, blockedByTransitive, dependencyChain dynamically

  Action: Update schema to remove dependencies, keep only blockedBy.
