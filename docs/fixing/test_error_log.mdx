üî¥ **ISSUE-003: Backend tasks get wrong ID prefix**
- **Affects**: Creating tasks with `-pl backend` or `--platform backend`
- **Expected**: Tasks should get ID prefix `be-XXXX`
- **Actual**: Tasks get ID prefix `fe-XXXX`
- **Example**:
  ```bash
  pnpm task-manager -p test -pl backend task create --name "Test" --level high --component X
  # Creates: fe-0001 (WRONG - should be be-0001)
  ```
- **Impact**: Backend tasks cannot be listed or accessed correctly
- **Test Case**: TC-716 (backend task creation)
- **Evidence**:
  - File: `.pland/test-backend/backend-tasks.jsonl` contains task with `id: "fe-0001"`
  - `task list` shows "No tasks found"
  - `task detail be-0001` shows "Task not found"
- **Severity**: **CRITICAL** - Backend functionality completely broken
- **Workaround**: None - cannot use backend platform
- **Status**: **BLOCKING** - needs immediate fix

üî¥ **ISSUE-005: --notReady flag doesn't work**
- **Affects**: Creating tasks with `--notReady` flag
- **Expected**: Task should be marked as not ready (`ready: false`)
- **Actual**: Task is still created with `ready: true` status
- **Example**:
  ```bash
  pnpm task-manager -p test task create --name "Test" --level high --component X --notReady
  # Task shows "Status: Ready" (WRONG - should be "Pending" or "Not Ready")
  ```
- **Impact**: Cannot manually mark tasks as not ready during creation
- **Test Case**: TC-246 (notReady flag)
- **Evidence**: Task detail shows "Status: Ready" despite `--notReady` flag
- **Severity**: **HIGH** - Feature not working as documented
- **Workaround**: Create task then use `update --notReady`
- **Status**: Needs fix in task creation logic
